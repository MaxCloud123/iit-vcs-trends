Here's a small timeline of major VCSs:

1972: Source Code Control System
The very first VCS, which started the idea of tracking changes and was made by Bell Labs.

1982: Revision Control System
Created by Walter Tichy at Purdue University, it automated tracking changes by using delta storage.

1986: Concurrent Versions System
A "front-end" to Revision Control System and the first VCS to introduce collaboration to version control.
Developed by Dick Grune and was the first major VCS to become widely used in 90's.

2000: Subversion
A successor to Concrrent Versions System, it introduced atomic commits, improved file renaming and network operations handling.
By 2006 it became one of the most used VCS.

2005: Git
Created by the legendary Linus Torvalds, it introduced distributed VCS and quickly became the number one Version Control System used by almost everyone till this day.
